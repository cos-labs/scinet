Frequently Asked Questions

Q: How does Citelet work?
A: Whenever you view an article on a website, Citelet first determines if it is from a publisher from which metadata can be extracted. If it is, Citelet determines whether the dataset already has the article. If it does not, then Citelet captures the citation and reference information and sends it all to our system for processing and storage.

Q: Why does Citelet look for a publisher on each page? Can’t you just look at the URL?
A: Unfortunately, it’s not quite this simple. Many articles are behind paywalls and only available through an institutional subscription or via direct purchase. Managing this process involves changes to the URL that you see. As a result, Citelet must check the source code of each page for something unique to identify a publishing source.

Q: What information does Citelet gather?
A: Each scientific article contains a large set of metadata including the title, author(s), journal name, publication data, and citations. Citelet’s goal is to collect as much of this metadata as possible before sending it to our system.

Q: What about the text and figures?
A: Citelet does not make any attempt to gather this or any other copyrighted data. Our goal is solely to collect the metadata about an article to help create a free, open, and comprehensive dataset.

Q: How does Citelet protect my privacy and identity?
A: The only information Citelet gathers from the user is the IP address of the machine they were using. This is simply a usage statistic that will help us determine where in the world Citelet is making the biggest impact. This information will be protected and not used for any other purpose.

Q: How many publishing sources does Citelet currently support?
A: We currently support 27 publishing sources.
